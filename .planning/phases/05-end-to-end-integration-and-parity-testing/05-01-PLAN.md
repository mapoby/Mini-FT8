No new code is expected -- Phase 3's CAT command implementation and Phase 4's audio negotiation are both already complete and hardware-validated in isolation (03-02-SUMMARY.md, 04-02-SUMMARY.md). Claude's Discretion (explicitly flagged, not silently assumed): ROADMAP Success Criterion 1 says "against a real or loopback contact." Either satisfies E2E-01:
  (a) Real on-air contact: the FTX-1 completes a genuine FT8 QSO with another station on the air.
  (b) Loopback / two-radio setup: the FTX-1 transmits into a second receiver (another Mini-FT8 instance, a second radio, or WSJT-X on a separate rig/SDR) that decodes and replies, completing the QSO exchange without depending on real-world band conditions or another operator's availability.

If this checkpoint surfaces a genuine code defect (not a methodology/environment issue), it must be handled as an explicit gap-closure fix task in a follow-up plan -- mirroring 03-01 Task 3's gap-closure addendum pattern -- not worked around silently or deferred without a trace.
