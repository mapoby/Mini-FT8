<objective>
Validate on the physical FTX-1 that the CAT command implementation from plan 03-01 actually controls the radio. This phase's success criteria are hardware-verifiable per ROADMAP and cannot be proven by code review — mirroring Phase 2's 02-03 hardware checkpoint.

The single highest-risk item is Open Question 1 from 03-RESEARCH.md: whether `TX1;`/`TX0;` sent over CAT-1 (this project's only open port, per the locked single-port decision) actually keys the radio, since the CAT manual's port-function table associates "TX Controls" with CAT-2. If CAT-1 cannot key the radio, this is a roadmap-level architectural question that MUST be escalated to the user — not silently worked around by opening CAT-2 (explicitly Out of Scope).

Purpose: Confirm SYNC-01/02/03 and PTT-01/02 behave correctly against real hardware, and resolve the CAT-1 PTT-scoping open question.
Output: 03-02-SUMMARY.md recording pass/fail per success criterion and the CAT-1 PTT resolution.
</objective>
