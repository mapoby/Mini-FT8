<objective>
Give the FTX-1 backend a real CAT command implementation on top of the CP210x CAT-1 port that Phase 2 already opens. Two independent code changes:

1. Transport tier (`stream_uac.cpp`): configure CAT-1 line coding to 38400 8N1 at port-open time — a hard prerequisite Phase 2 did not wire, without which every CAT command risks being framed at the wrong baud rate.
2. Backend tier (`radio_control_ftx1.cpp`): replace the Phase 1 stubs (which return `ESP_ERR_INVALID_STATE`) with real `FA`/`MD`/`TX` command encoding, using the FTX-1's verified Yaesu field widths — NOT the Kenwood-style widths QMX/QDX use.

Purpose: Satisfy SYNC-01/02/03 (frequency, DATA-U mode, post-TX restore) and PTT-01/02 (TX1;/TX0; keying, drain-before-unkey) in code. Hardware validation of these on the physical FTX-1 is done in the follow-on checkpoint plan (03-02).
Output: A firmware build in which selecting FTX-1 and syncing/keying issues correct CAT bytes over the already-open CP210x handle.
</objective>
