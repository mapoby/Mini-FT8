<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 1: Flash and verify FTX-1 CP210x CAT bring-up on physical hardware</name>
  <what-built>
    Plan 01 added the `usb_host_cp210x_vcp` dependency and routed FTX-1 to its own
    `UAC_PROFILE_FTX1`/`AUDIO_SOURCE_FTX1_CP210X` (fixing the Phase-1 QMX-alias placeholder).
    Plan 02 added `cp210x_try_open()` (opens the CP210x CAT-1 port and deasserts RTS/DTR),
    hard-gated `cdc_try_open()` to the QMX profile only, broadened the CDC-ACM install gate to
    the FTX-1 profile, and wired `ftx1_ready()` to `cat_cdc_ready()`. All code is committed and
    the project builds clean under `-Werror`.
  </what-built>
  <how-to-verify>
    Prerequisites: run `idf.py reconfigure` (fetches the cp210x component), then `idf.py build flash monitor`.

    1. ENUMERATION/OPEN: With FTX-1 selected as the radio, plug the FTX-1 into the Cardputer.
       Confirm the monitor log shows `CDC-ACM driver installed` exactly once, then
       `CP210x opened (FTX-1 CAT-1 iface 0); RTS/DTR deassert: ESP_OK`. Confirm the
       `cdc_acm_host_desc_print()` dump appears. RELIABILITY: cold-boot and replug at least
       3 times; the port must open every time (no stuck `ESP_ERR_NOT_FOUND` loop).
    2. PID/INTERFACE CONFIRMATION: In the descriptor dump, confirm the device VID is 0x10C4 and
       the PID matches `CP2105_PID` (0xEA70). Confirm interface index 0 is the CAT-1 (Enhanced
       COM) port. If EITHER differs: report the actual PID and which interface index is CAT-1 —
       the executor will apply a one-line change to `cp210x_try_open()` (switch to
       `CP210X_PID_AUTO` for a PID mismatch, or change `interface_idx` for an index mismatch) and
       you re-verify. This is the expected path if the research assumptions (A1/A2) were wrong.
    3. NO SPURIOUS PTT (CAT-03): Watch the FTX-1's own display during open and across all replug
       cycles — it must NOT show TX/keyup/PTT. The RTS/DTR deassert must prevent any keydown.
    4. NO MISCLAIM (CAT-02): Confirm the log shows NO blind interface-scan attempts or CDC-ACM
       open errors against the FTX-1's vendor-class interface (no `CDC open ... iface N failed`
       spam walking iface 0..11). Expected benign noise: a mic/UAC "Format not supported" style
       line may appear because Phase 2 does not negotiate FTX-1 audio yet (RESEARCH.md Open
       Question 2) — this is NOT a regression; do not chase it.
    5. QMX NO-REGRESSION: Switch the radio to QMX, plug in a QMX, and confirm it still enumerates
       and opens via the existing CDC-ACM path (`CDC-ACM opened (QMX iface 0 ...)`), unchanged.
  </how-to-verify>
  <resume-signal>
    Type "approved" if all five checks pass. Otherwise report which check failed with the relevant
    log lines (for check 2, include the observed VID/PID and CAT-1 interface index so the one-line
    correction can be applied).
  </resume-signal>
  <acceptance_criteria>
    - Monitor log shows a single `CDC-ACM driver installed` and `CP210x opened (FTX-1 CAT-1 iface 0)` with `RTS/DTR deassert: ESP_OK`, reproducibly across >= 3 cold-boot/replug cycles.
    - Observed VID/PID and CAT-1 interface index either match the assumed CP2105_PID/index 0, or the one-line correction has been applied and re-verified.
    - FTX-1 display shows no spurious PTT/keyup during or after open.
    - No CDC-ACM blind-scan/misclaim errors against the FTX-1 vendor-class interface in the log.
    - QMX still enumerates and opens with no regression.
  </acceptance_criteria>
  <done>All four phase success criteria are confirmed on physical hardware; any PID/interface-index correction is committed and re-verified.</done>
