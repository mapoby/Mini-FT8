# Getting Started

Mini-FT8 is an ESP-IDF firmware image for the M5Stack Cardputer ADV (ESP32-S3). There is no
package manager and no "dev server" — getting started means installing the ESP-IDF v5.5.1
toolchain, building the firmware, and flashing it to the physical device.

## Prerequisites

- **M5Stack Cardputer ADV** (K132-ADV) — required. See the Hardware section of the root
  [README.md](../README.md) for purchase links and optional accessories (GPS, DS3231 RTC,
  KH1 cabling, FTX-1 USB support).
- **ESP-IDF v5.5.1** toolchain, including the `xtensa-esp32s3-elf` compiler toolchain.
  Install via the [official Espressif installer](https://docs.espressif.com/projects/esp-idf/en/v5.5.1/esp32s3/get-started/index.html)
  (Windows: `esp-idf` installer that provisions `C:\Espressif\esp-idf-v5.5.1`; Linux/macOS:
  the `install.sh` / `install.fish` bootstrap scripts from the same ESP-IDF release).
  `sdkconfig` in this repo pins `IDF_TARGET_ARCH_XTENSA` for ESP32-S3 and was generated
  against ESP-IDF 5.5.1 — other 5.x versions may not be binary-compatible with the checked-in
  `sdkconfig`.
- **Python 3.8+** — bundled and managed by the ESP-IDF installer/export script; no separate
  install is needed if you used the official installer.
- **CMake 3.16+** and **Ninja** — also provisioned by the ESP-IDF installer.
- A **USB cable** to connect the Cardputer ADV to your PC for flashing (USB-C).
- Internet access on first build — the ESP-IDF Component Manager auto-fetches dependencies
  listed in `main/idf_component.yml` (pinned in `dependencies.lock`) into `managed_components/`
  on the first `idf.py build`. These include `espressif/usb_host_uac`,
  `espressif/usb_host_cdc_acm`, `espressif/usb_host_cp210x_vcp`, `espressif/esp_codec_dev`,
  and `lvgl/lvgl`.

## Installation Steps

1. Clone the repository:
   ```bash
   git clone <repository-url>
   cd Mini-FT8
   ```
2. Install ESP-IDF v5.5.1 following Espressif's official installer for your platform, if not
   already installed.
3. Load the ESP-IDF environment into your shell (do this in every new shell session before
   building):

   **Windows (PowerShell):**
   ```powershell
   . C:\Espressif\esp-idf-v5.5.1\export.ps1
   ```

   **Windows (cmd.exe):**
   ```bat
   call C:\Espressif\esp-idf-v5.5.1\export.bat
   ```

   **Linux / macOS:**
   ```bash
   . $HOME/esp/esp-idf/export.sh
   ```
   <!-- VERIFY: exact Linux/macOS ESP-IDF install path — depends on where the installer was
   pointed; `$HOME/esp/esp-idf` is the ESP-IDF documentation's default convention, not
   confirmed against this repository. -->

## First Run

1. Build the firmware:
   ```bash
   idf.py build
   ```
   The first build fetches all managed components (see Prerequisites) and produces
   `build/mini_ft8.bin` plus a merged image, `build/MiniFT8_Merged_Auto.bin`, generated by
   the `esptool merge_bin` post-build step defined in the root `CMakeLists.txt`.
2. Connect the Cardputer ADV via USB and identify its serial port (Windows: `COMx` in Device
   Manager; Linux: `/dev/ttyACM*` or `/dev/ttyUSB*`; macOS: `/dev/cu.usbmodem*`).
3. Flash it:
   ```bash
   idf.py -p <PORT> flash
   ```
   Replace `<PORT>` with your device's port (e.g., `COM12` on Windows — this exact command
   was verified in this repo's own `build.ps1`/`build_and_flash.bat` scripts).
4. Watch the device boot. On first boot with the FATFS-only partition table, the internal
   3 MB FATFS partition is formatted automatically (see `partitions.csv`). Once booted, the
   Cardputer shows the RX screen; press the mode keys described in the root
   [README.md](../README.md) Quick Mode Map to explore the UI.
5. Optional: stream live logs instead of/after flashing:
   ```bash
   idf.py -p <PORT> monitor
   ```

## Common Setup Issues

- **`idf.py` not found / "IDF_PATH not set".** The ESP-IDF environment must be re-loaded
  (`export.ps1` / `export.bat` / `export.sh`) in every new terminal session — it is not
  persisted globally. If you see this, re-run the export step from Installation Steps 3.
- **First build is slow or fails with a network error.** The first `idf.py build` fetches
  all `managed_components/` dependencies pinned in `dependencies.lock`
  (`espressif/usb_host_uac`, `espressif/usb_host_cdc_acm`, `espressif/usb_host_cp210x_vcp`,
  `espressif/esp_codec_dev`, `lvgl/lvgl`, etc.) from the ESP-IDF Component Registry — this
  requires internet access. Subsequent builds are offline once `managed_components/` is
  populated.
- **`idf.py -p <PORT> flash` can't find/open the port, or the port disappears.** Confirm the
  Cardputer ADV is connected directly (not through the same physical port a USB host device
  such as a radio is using) and that no other program (serial monitor, Arduino IDE, etc.) has
  the port open. If flashing over the same USB-C port used later for a USB-host-mode radio
  connection (QMX/FTX-1), disconnect the radio during flash/monitor and reconnect it only
  after the firmware is running.
  <!-- VERIFY: whether the Cardputer ADV's flashing/debug USB link and its USB-host radio
  connection share the same physical port or are independent — not confirmed from this
  repository's source or docs. -->
- **Wrong ESP-IDF version.** The checked-in `sdkconfig` was generated against ESP-IDF 5.5.1
  (`Automatically generated file... Espressif IoT Development Framework (ESP-IDF) 5.5.1`).
  Building with a different major/minor ESP-IDF release may produce config warnings or
  build errors; install exactly v5.5.1 if you hit unexpected `idf.py build` failures.
